# Source Control

## Definition

Source control, also known as version control or revision control, is a system for tracking changes in source code over time. It allows multiple users to make edits to the same codebase simultaneously without interfering with each other's work.

## Key Principles

1. **Version Tracking**: Each change made to the codebase is recorded and can be reverted at any point.
2. **Branching and Merging**: Developers can create branches of the source code for their own changes, which can then be merged back into the main branch when ready.
3. **Collaboration**: Multiple developers can work on different parts of a project without conflicting with each other's changes.
4. **History Management**: A detailed history of all changes is maintained, making it easy to trace and revert to previous versions if necessary.

## Examples

1. **Git**: One of the most widely used version control systems. It is distributed, meaning that every developer has a complete copy of the repository on their local machine.
2. **SVN (Subversion)**: A centralized version control system where all changes are made through a central server.
3. **Mercurial**: Another distributed version control system that is similar to Git but with a simpler interface.
